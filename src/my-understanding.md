ทำ navbar ก่อน เพราะมันต้องเอาไปใช้กับทุกหน้า
สร้างปุ่มกด 2 ปุ่ม ในหน้า Home
ใช้ useState เพื่อเก็บค่าตอน user คลิกที่ปุ่ม จะได้เรนเดอร์ Table สลับกับ Form ได้ภายในหน้า Home หน้าเดียว
สร้างตัวแปร Sector เพื่อใช้เก็บสถานะ ถ้ากดปุ่มซ้ายจะเซตเป็น "user" ถ้ากดปุ่มขวาจะเซตเป็น "admin"

สร้างตารางใน UserTable.jsx
ไปหน้า Home.jsx แล้วดึงตารางเข้ามา
สร้าง State สำหรับเก็บรายชื่อพนักงาน แล้วส่งข้อมูลไปให้ UserTable และ AdminForm ผ่านทาง Props ทำแบบนี้จะทำให้ข้อมูลทั้ง 2 หน้าตรงกัน

ใช้ useEffect คู่กับ fetch เพื่อดึงข้อมูลจาก API แล้วเอาข้อมูลที่ได้ไปเก็บในตัวแปร data (ใช้ useEffect แก้ปัญหา ให้ดึงข้อมูลแค่ครั้งเดียว ป้องกันไม่ให้ดึงข้อมูลวนลูป)

ใช้การดึงข้อมูลแบบ Asynchronous (ใช้ async/await) เพื่อให้หน้าเว็บยังทำงานต่อไปได้ ถ้ารอให้โหลดครบก่อน หน้าเว็บอาจจะค้าง เพราะข้อมูลอาจจะเยอะมาก

สร้าง form ที่ เพิ่ม/ลบ user ได้ ใน AdminForm.jsx

ไปหน้า Home.jsx แล้ว import หน้า AdminForm มาใช้

ทำให้ฟังก์ชัน handleCreate ทำงานแบบ Real-time เพื่อให้โชว์ข้อมูลที่เซฟเข้าไป

ทำฟังก์ชันในปุ่มเซฟที่อยู่ในหน้า AdminForm พอกด Save ฟอร์มจะเอาข้อมูลจาก State ของ Input (ที่มีชื่อ นามสกุล ตำแหน่ง) ส่งกลับไปให้ฟังก์ชัน onCreate ที่รับมาจากตัวพ่อในหน้า Home

ที่ไฟล์ตัวพ่อ (Home.jsx) จะ ใช้ fetch() แบบ POST เพื่อส่งข้อมูลไปเก็บไว้ที่ฐานข้อมูล MockAPI

การอัปเดตหน้าจอแบบ Real-time: หลังจากส่งข้อมูลสำเร็จ ใช้ fetchMembers เพื่อไปดึงข้อมูลล่าสุดจาก Server มาโชว์ในตาราง user จะเห็นว่าข้อมูลขึ้นมาแล้ว จะได้ไม่ต้องกด Refresh หน้าเว็บ

ทำให้ปุ่ม Delete เมื่อกดแล้ว จะใช้ฟังก์ชั่น onDelete โดยจะส่ง ID กลับไปให้ตัวพ่อ เพื่อยิง api แบบ DELETE Method ไปที่ URL ที่ระบุ ID นั้น ๆ (เช่น /members/1 เพราะเราผูกปุ่ม Delete ไว้กับ item.id ของเมมเบอร์นั้นๆ ไว้แล้ว)

พอ Server ลบข้อมูลสำเร็จแล้ว เราจะใช้ fetchMembers เพื่อโหลดข้อมูลชุดใหม่ที่ไม่มีคนที่เราลบออกมาแสดงผล
